# WhisperBar — 2026-03-09 更新日志

## 一、Whisper 参数调优（减少"未检测到语音"）

**对应问题**：用户明确说了话，但 App 显示"未检测到语音"

**改动文件**：`TranscriptionEngine.swift`

**改动内容**：在 `DecodingOptions` 中调整三个关键参数：

```swift
let options = DecodingOptions(
    ...
    compressionRatioThreshold: 3.0,  // 默认 2.4 → 对噪音环境更宽容
    logProbThreshold: -1.5,          // 默认 -1.0 → 接受置信度较低的结果
    noSpeechThreshold: 0.3           // 默认 0.6 → 更努力地尝试识别语音
)
```

**原理**：
- `noSpeechThreshold` 从 0.6 降到 0.3：Whisper 内部计算"无语音概率"，超过阈值就返回空结果。降低阈值让模型在不确定时也尝试输出，而不是直接放弃
- `compressionRatioThreshold` 从 2.4 提高到 3.0：压缩比过高通常意味着输出质量差（如重复文本），放宽阈值对嘈杂环境更友好
- `logProbThreshold` 从 -1.0 降到 -1.5：接受平均 log 概率更低的结果，避免因模型不够自信而丢弃有效转写

## 二、音频峰值归一化（提升小声录音识别率）

**对应问题**：声音小或距离麦克风较远时识别准确率下降

**改动文件**：`TranscriptionEngine.swift`

**改动内容**：新增 `normalizeAudio(at:)` 静态方法，在录音完成后、送入 Whisper 之前自动归一化音量：

```swift
private nonisolated static func normalizeAudio(at url: URL) -> URL {
    // 1. 用 AVAudioFile 读取 WAV 文件
    // 2. 用 vDSP_maxmgv 找到峰值振幅
    // 3. 如果峰值 < 0.5 (-6dB)，按比例放大到 -3dB (0.707)
    // 4. 放大倍数上限 10×，避免纯噪音被过度放大
    // 5. 写入新的临时 WAV 文件
}
```

**处理逻辑**：
- 峰值 < 0.0001 → 静音，跳过
- 峰值 0.0001 ~ 0.5 → 归一化到 -3dB
- 峰值 ≥ 0.5 → 已经足够响，跳过

**技术细节**：
- 使用 `Accelerate/vDSP` 框架做向量运算，几乎零延迟
- 归一化文件使用 UUID 命名避免冲突
- 原始文件和归一化文件均在 `defer` 中清理

## 三、词汇注入 Whisper Prompt（核心修复）

**对应问题**：用户在词汇管理中添加的内容对识别结果无帮助

**根本原因**：之前词汇只用于转写完成后的字符串替换（事后补救），完全没有参与 Whisper 的解码过程。模型在推理时根本不知道用户的自定义词汇。

**改动文件**：`TranscriptionEngine.swift`

**改动内容**：动态构建包含词汇的 prompt 并编码为 `promptTokens`：

```swift
var prompt = "以下是普通话的句子。"
if let tokenizer = kit.tokenizer,
   let terms = vocabularyStore?.terms, !terms.isEmpty {
    let names = terms.map { $0.correct }
    var suffix = "涉及的专有名词："
    for (i, name) in names.enumerated() {
        let candidate = suffix + name + (i < names.count - 1 ? "、" : "。")
        if tokenizer.encode(text: prompt + candidate).count > 200 { break }
        suffix = candidate
    }
    if suffix != "涉及的专有名词：" { prompt += suffix }
}
let promptTokens = kit.tokenizer?.encode(text: prompt)
```

生成的 prompt 示例：`"以下是普通话的句子。涉及的专有名词：OpenAI、Claude、李明。"`

**token 限制保护**：Whisper 的 prompt token 窗口为 224 tokens。逐词编码并检查长度，超过 200 tokens 时停止添加，避免超限。

**双层机制**：词汇现在同时参与两个环节：
1. **解码前**：注入 prompt，引导 Whisper 正确输出这些词汇
2. **解码后**：字符串替换兜底，修正残留错误

## 四、词汇替换改为大小写不敏感

**对应问题**：Whisper 输出的大小写不可预测，导致精确匹配失败

**改动文件**：`TranscriptionEngine.swift`

**改动内容**：

```swift
// 之前（大小写敏感）
text = text.replacingOccurrences(of: wrong, with: correct)

// 之后（大小写不敏感）
text = text.replacingOccurrences(of: wrong, with: correct, options: .caseInsensitive)
```

**效果**：别名 `open ai` 现在可以匹配 `Open AI`、`OPEN AI`、`open ai` 等所有变体。

## 五、浮动面板 UI 修复

**对应问题**：
1. 圆角内容外面有一个方形边框，不美观
2. 面板不能拖动

**改动文件**：`AppDelegate.swift`

**改动内容**：

```swift
panel.hasShadow = false                // 去掉窗口级矩形阴影
panel.isMovableByWindowBackground = true // 拖动面板任意非按钮区域即可移动
```

**原理**：
- NSPanel 的 `hasShadow = true` 会生成矩形的窗口级阴影，叠在 SwiftUI 的圆角 `.shadow()` 上形成方形外框。关闭后只保留 SwiftUI 的圆角阴影
- `isMovableByWindowBackground` 让面板中非交互区域（毛玻璃背景、状态文字等）成为拖拽手柄，按钮仍正常响应点击
