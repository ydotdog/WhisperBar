# WhisperBar — 2026-03-08 更新日志

## 本次解决的两个痛点

### 痛点 1：中文语音偶尔被整段输出为英文

**现象**
用户说中文时，转写结果偶尔变成英文（不是夹杂英文词，而是整段中文含义被翻译成英文输出）。此前已通过 Bug 11 修复了 `language: "zh"` 强制中文模式，但问题仍偶发。

**根本原因**

之前的 `DecodingOptions` 只设置了 `language: "zh"` 和 `usePrefillPrompt: true`，存在两个不足：

1. **`detectLanguage` 未显式关闭**：该参数默认为 `nil`，WhisperKit 内部会根据逻辑决定是否仍执行语言检测，在某些情况下可能覆盖手动指定的 `language: "zh"`
2. **缺少 initial prompt 偏置**：Whisper large-v3-turbo 在海量多语言数据上训练，英文数据占比最大，模型对英文有很强的先验偏好。仅靠 `<|zh|>` 前缀 token 是一个弱约束，当语音较短、技术术语较多或发音接近英文音素时，解码器可能忽略语言 token 直接输出英文

**解决方案**

在 `TranscriptionEngine.swift` 的 `transcribeFile` 中修改 `DecodingOptions`：

```swift
let promptTokens = kit.tokenizer?.encode(text: "以下是普通话的句子。")
let options = DecodingOptions(
    task: .transcribe,
    language: "zh",
    usePrefillPrompt: true,
    detectLanguage: false,        // 显式关闭自动语言检测
    promptTokens: promptTokens    // 注入中文 initial prompt
)
```

**原理**

- `detectLanguage: false`：彻底阻止 WhisperKit 在解码前执行语言检测，避免检测结果覆盖 `language: "zh"`
- `promptTokens`：利用 Whisper 的 initial prompt 机制，将中文提示文本编码为 token 序列注入解码器上下文。解码器在生成每个新 token 时会 attend to 所有之前的 token（包括这些中文 prompt token），条件概率分布被强力偏向中文输出。这是 Whisper 官方推荐的控制输出语言的方法
- 中英混合（code-switching）不受影响——Whisper 在中文模式下本身支持保留嵌入的英文单词

### 痛点 2：必须手动 Cmd+V 粘贴转写文本

**现象**
转写完成后文字仅复制到剪贴板，用户必须手动按 Cmd+V 粘贴到光标位置。

**解决方案**

在 `TranscriptionEngine.swift` 中新增 CGEvent 模拟 Cmd+V 按键功能：

```swift
private static func simulatePaste() {
    let src = CGEventSource(stateID: CGEventSourceStateID.hidSystemState)
    let vKey: CGKeyCode = 9 // kVK_ANSI_V

    let down = CGEvent(keyboardEventSource: src, virtualKey: vKey, keyDown: true)
    down?.flags = CGEventFlags.maskCommand
    down?.post(tap: CGEventTapLocation.cgSessionEventTap)

    let up = CGEvent(keyboardEventSource: src, virtualKey: vKey, keyDown: false)
    up?.flags = CGEventFlags.maskCommand
    up?.post(tap: CGEventTapLocation.cgSessionEventTap)
}
```

转写完成后的逻辑改为：

```swift
if AXIsProcessTrusted() {
    // 有辅助功能权限 → 自动粘贴到光标位置
    DispatchQueue.main.asyncAfter(deadline: .now() + 0.05) {
        Self.simulatePaste()
    }
    statusText = "已粘贴到光标位置"
} else {
    // 无权限 → 退化为仅复制到剪贴板
    statusText = "已复制到剪贴板"
}
```

**原理**

- `CGEvent` 创建虚拟键盘事件（V 键 + Command 修饰键），注入系统事件流
- WhisperBar 面板是 `nonactivatingPanel`（不抢焦点），录音和转写期间光标仍在之前的应用中，模拟的 Cmd+V 会发送到该应用
- 需要 macOS 辅助功能（Accessibility）权限，通过 `AXIsProcessTrusted()` 检查
- 未授权时优雅降级为原有行为（仅复制到剪贴板）
- 0.05 秒延迟确保剪贴板写入完成后再触发粘贴

**其他配套改动**

- 启动时通过 `checkAccessibility()` 检查辅助功能权限状态
- 面板底部提示文案改为"授权辅助功能可自动粘贴到光标位置"，点击"去授权"跳转系统设置
- 状态文案根据是否有权限动态显示"已粘贴到光标位置"或"已复制到剪贴板"
